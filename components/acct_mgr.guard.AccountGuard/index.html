acct_mgr.guard.AccountGuard

In package TracAccountManager

The AccountGuard component protects against brute-force attacks
on user passwords.

It does so by adding logging of failed login attempts,
account status tracking and administative user account locking.
Configurable time-locks with exponential lock time prolongation
allow to balance graceful handling of failed login attempts and
reasonable protection against attempted brute-force attacks.

Contains via Extension Points:

Options:

SectionNameTypeDefaultDocs
account-manager login_attempt_max_count IntOption 0 Lock user account after specified number of login attempts. Value zero means no limit.
account-manager user_lock_max_time IntOption 86400 Limit user account lock time to specified time (seconds). This is relevant only with user_lock_time_progression > 1.
account-manager user_lock_time IntOption 0 Drop user account lock after specified time (seconds). Value zero means unlimited lock time.
account-manager user_lock_time_progression Option 1 Extend user account lock time incrementally. This is based on logarithmic calculation and decimal numbers accepted: Value '1' means constant lock time per failed login attempt. Value '2' means double locktime after 2nd lock activation, four times the initial locktime after 3rd, and so on.
